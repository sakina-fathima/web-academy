Asynchronous Web

JavaScript is usually synchronous (one thing at a time). However, when we need to get data from a server, we must go Asynchronous to prevent the app from freezing.

1. Promises (The Handshake)

Definition

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation. It is a "placeholder" for a value that hasn't arrived yet. 

// A promise can be Pending, Fulfilled, or Rejected
getData()
  .then(data => console.log(data))
  .catch(err => console.error(err));

2. The Fetch API

Definition

The Fetch API is a modern interface that allows you to make HTTP requests to servers. It is the primary way we get data (like weather, prices, or users) into our applications. 

// Fetching user data from a placeholder API
fetch('https://api.example.com/user/1')
  .then(response => response.json())
  .then(json => console.log(json));

3. Async / Await

Definition

Async/Await is special syntax used to work with Promises in a more comfortable, "synchronous-looking" way. It makes code easier to read and maintain. 

// The modern standard for async logic
async function loadData() {
  const response = await fetch(url);
  const data = await response.json();
  return data;
}
