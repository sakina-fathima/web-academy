Optimization

Writing code that works is the first step. Writing code that is efficient, fast, and scalable is what defines a Senior Engineer.

1. Code Complexity (Big O)

Definition

Big O Notation is a mathematical notation that describes the limiting behavior of a function when the argument tends towards a particular value or infinity. In web dev, it measures how time or space requirements grow as data increases. 

// O(1) - Constant Time (Fast)
const firstItem = arr[0];

// O(n) - Linear Time (Grows with data)
arr.forEach(item => console.log(item));

2. Debouncing

Definition

Debouncing is a programming practice used to ensure that time-consuming tasks do not fire so often that they make the web page lag. It limits the rate at which a function can fire. 

// Delaying a search until typing stops
let timer;
input.addEventListener('input', () => {
  clearTimeout(timer);
  timer = setTimeout(() => fetchResults(), 500);
});

3. Clean Code & Refactoring

Definition

Refactoring is the process of restructuring existing computer code—changing the factoring—without changing its external behavior. The goal is to improve readability and reduce complexity. 

// Before: Hard to read
if(user.active && user.age > 18) { ... }

// After: Semantic & Clean
const canAccess = user.active && user.age > 18;
if(canAccess) { ... }
